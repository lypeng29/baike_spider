<div class="para" label-module="para">PHP（外文名:PHP: Hypertext Preprocessor，中文名：“<a target=_blank href="/item/%E8%B6%85%E6%96%87%E6%9C%AC">超文本</a><a target=_blank href="/item/%E9%A2%84%E5%A4%84%E7%90%86%E5%99%A8">预处理器</a>”）是一种通用<a target=_blank href="/item/%E5%BC%80%E6%BA%90">开源</a><a target=_blank href="/item/%E8%84%9A%E6%9C%AC%E8%AF%AD%E8%A8%80">脚本语言</a>。<a target=_blank href="/item/%E8%AF%AD%E6%B3%95">语法</a>吸收了<a target=_blank href="/item/C%E8%AF%AD%E8%A8%80">C语言</a>、<a target=_blank href="/item/Java">Java</a>和<a target=_blank href="/item/Perl">Perl</a>的特点，利于学习，使用<a target=_blank href="/item/%E5%B9%BF%E6%B3%9B">广泛</a>，主要适用于<a target=_blank href="/item/Web">Web</a>开发领域。PHP 独特的<a target=_blank href="/item/%E8%AF%AD%E6%B3%95">语法</a>混合了<a target=_blank href="/item/C">C</a>、<a target=_blank href="/item/Java">Java</a>、<a target=_blank href="/item/Perl">Perl</a>以及<a target=_blank href="/item/PHP">PHP</a>自创的语法。它可以比<a target=_blank href="/item/CGI">CGI</a>或者<a target=_blank href="/item/Perl">Perl</a>更快速地执行<a target=_blank href="/item/%E5%8A%A8%E6%80%81%E7%BD%91%E9%A1%B5">动态网页</a>。用PHP做出的<a target=_blank href="/item/%E5%8A%A8%E6%80%81%E9%A1%B5%E9%9D%A2">动态页面</a>与其他的<a target=_blank href="/item/%E7%BC%96%E7%A8%8B%E8%AF%AD%E8%A8%80">编程语言</a>相比，<a target=_blank href="/item/PHP/9337" data-lemmaid="9337">PHP</a>是将<a target=_blank href="/item/%E7%A8%8B%E5%BA%8F">程序</a>嵌入到<a target=_blank href="/item/HTML">HTML</a>（<a target=_blank href="/item/%E6%A0%87%E5%87%86%E9%80%9A%E7%94%A8%E6%A0%87%E8%AE%B0%E8%AF%AD%E8%A8%80">标准通用标记语言</a>下的一个应用）文档中去执行，执行效率比完全生成<a target=_blank href="/item/HTML">HTML</a>标记的<a target=_blank href="/item/CGI/607810" data-lemmaid="607810">CGI</a>要高许多；PHP还可以执行<a target=_blank href="/item/%E7%BC%96%E8%AF%91">编译</a>后代码，编译可以达到<a target=_blank href="/item/%E5%8A%A0%E5%AF%86">加密</a>和<a target=_blank href="/item/%E4%BC%98%E5%8C%96">优化</a>代码运行，使代码运行更快。</div>
</div>

</div><div class="para-title level-2" label-module="para-title">
<h2 class="title-text"><span class="title-prefix">PHP</span>发展</h2>
</div>
<div class="para" label-module="para"><div class="lemma-picture text-pic layout-right" style="width:199px; float: right;">
<a class="image-link" nslog-type="9317" 
			href="/pic/PHP/9337/0/814b07d8dbda807632fa1c0c?fr=lemma&ct=single" target="_blank"
		title="PHP安装" style="width:199px;height:220px;">
<img  class="lazy-img" src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAMAAAAoyzS7AAAAGXRFWHRTb2Z0d2FyZQBBZG9iZSBJbWFnZVJlYWR5ccllPAAAAAZQTFRF9fX1AAAA0VQI3QAAAAxJREFUeNpiYAAIMAAAAgABT21Z4QAAAABJRU5ErkJggg==" data-src="https://gss3.bdstatic.com/7Po3dSag_xI4khGkpoWK1HF6hhy/baike/s%3D220/sign=d79afbc365380cd7e21ea5ef9145ad14/f11f3a292df5e0fefec18d725c6034a85fdf72e3.jpg"  alt="PHP安装" style="width:199px;height:220px;"/>
</a>
<span class="description">
PHP安装
</span>
</div>PHP原始为Personal Home Page的缩写，已经正式更名为 &quot;PHP: Hypertext Preprocessor&quot;。注意不是“Hypertext Preprocessor”的缩写，这种将名称放到定义中的写法被称作<a target=_blank href="/item/%E9%80%92%E5%BD%92%E7%BC%A9%E5%86%99">递归缩写</a>。PHP于1994年由<a target=_blank href="/item/Rasmus%20Lerdorf">Rasmus Lerdorf</a>创建，刚刚开始是<a target=_blank href="/item/Rasmus%20Lerdorf">Rasmus Lerdorf</a>为了要<a target=_blank href="/item/%E7%BB%B4%E6%8A%A4">维护</a>个人网页而制作的一个简单的用<a target=_blank href="/item/Perl">Perl</a>语言编写的程序。这些工具程序用来显示 Rasmus Lerdorf 的个人履历，以及统计网页流量。后来又用<a target=_blank href="/item/C%E8%AF%AD%E8%A8%80">C语言</a>重新编写，包括可以访问<a target=_blank href="/item/%E6%95%B0%E6%8D%AE%E5%BA%93">数据库</a>。他将这些程序和一些<a target=_blank href="/item/%E8%A1%A8%E5%8D%95">表单</a>直译器整合起来，称为 PHP/FI。PHP/FI 可以和<a target=_blank href="/item/%E6%95%B0%E6%8D%AE%E5%BA%93">数据库</a>连接，产生简单的<a target=_blank href="/item/%E5%8A%A8%E6%80%81%E7%BD%91%E9%A1%B5">动态网页</a>程序。</div>
<div class="para" label-module="para">在1995年以Personal Home Page Tools (PHP Tools) 开始对外发表第一个版本，Lerdorf写了一些介绍此程序的文档。并且发布了PHP1.0！在这的版本中，提供了访客<a target=_blank href="/item/%E7%95%99%E8%A8%80%E6%9C%AC">留言本</a>、访客计数器等简单的功能。以后越来越多的网站使用了PHP，并且强烈要求增加一些特性。比如<a target=_blank href="/item/%E5%BE%AA%E7%8E%AF%E8%AF%AD%E5%8F%A5">循环语句</a>和数组<a target=_blank href="/item/%E5%8F%98%E9%87%8F">变量</a>等等；在新的成员加入开发行列之后，Rasmus Lerdorf 在1995年6月8日将 PHP/FI 公开发布，希望可以透过社群来加速程序开发与寻找错误。这个发布的版本命名为 PHP 2，已经有 PHP 的一些雏型，像是类似 <a target=_blank href="/item/Perl">Perl</a>的变量命名方式、表单处理功能、以及嵌入到 HTML 中执行的能力。程序语法上也类似 Perl，有较多的限制，不过更简单、更有弹性。PHP/FI加入了对MySQL的支持，从此建立了PHP在动态网页开发上的地位。到了1996年底，有15000个网站使用 PHP/FI。<div class="lemma-picture text-pic layout-right" style="width:215px; float: right;">
<a class="image-link" nslog-type="9317" 
			href="/pic/PHP/9337/0/a8ad941386c25393f6039e0e?fr=lemma&ct=single" target="_blank"
		title="ISAPI筛选器" style="width:215px;height:220px;">
<img  class="lazy-img" src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAMAAAAoyzS7AAAAGXRFWHRTb2Z0d2FyZQBBZG9iZSBJbWFnZVJlYWR5ccllPAAAAAZQTFRF9fX1AAAA0VQI3QAAAAxJREFUeNpiYAAIMAAAAgABT21Z4QAAAABJRU5ErkJggg==" data-src="https://gss1.bdstatic.com/-vo3dSag_xI4khGkpoWK1HF6hhy/baike/s%3D220/sign=9e6caf120fb30f24319aeb01f894d192/8601a18b87d6277f7bca364b28381f30e824fced.jpg"  alt="ISAPI筛选器" style="width:215px;height:220px;"/>
</a>
<span class="description">
ISAPI筛选器
</span>
</div></div>
<div class="para" label-module="para">在1997年，任职于 Technion IIT公司的两个<a target=_blank href="/item/%E4%BB%A5%E8%89%B2%E5%88%97">以色列</a>程序设计师：Zeev Suraski 和 Andi Gutmans，重写了 PHP 的剖析器，成为 PHP 3 的基础。而 PHP 也在这个时候改称为PHP：Hypertext Preprocessor。经过几个月测试，开发团队在1997年11月发布了 PHP/FI 2。随后就开始 PHP 3 的开放测试，最后在1998年6月正式发布 PHP 3。Zeev Suraski 和 Andi Gutmans 在 PHP 3 发布后开始改写PHP 的核心，这个在1999年发布的剖析器称为 Zend Engine，他们也在以色列的 Ramat Gan 成立了 Zend Technologies 来管理 PHP 的开发。</div>
<div class="para" label-module="para">在2000年5月22日，以Zend Engine 1.0为基础的PHP 4正式发布，2004年7月13日则发布了PHP 5，PHP 5则使用了第二代的Zend Engine。PHP包含了许多新特色，像是强化的<a target=_blank href="/item/%E9%9D%A2%E5%90%91%E5%AF%B9%E8%B1%A1">面向对象</a>功能、引入PDO（PHP Data Objects，一个存取数据库的延伸函数库）、以及许多效能上的增强。PHP 4已经不会继续<div class="lemma-picture text-pic layout-right" style="width:220px; float: right;">
<a class="image-link" nslog-type="9317" 
			href="/pic/PHP/9337/0/4e0b3ea4b3c538b49152ee0f?fr=lemma&ct=single" target="_blank"
		title="PHP" style="width:220px;height:172px;">
<img  class="lazy-img" src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAMAAAAoyzS7AAAAGXRFWHRTb2Z0d2FyZQBBZG9iZSBJbWFnZVJlYWR5ccllPAAAAAZQTFRF9fX1AAAA0VQI3QAAAAxJREFUeNpiYAAIMAAAAgABT21Z4QAAAABJRU5ErkJggg==" data-src="https://gss1.bdstatic.com/-vo3dSag_xI4khGkpoWK1HF6hhy/baike/s%3D220/sign=5e82320c78f0f736dcfe4b033a54b382/7af40ad162d9f2d3a2ca209ca9ec8a136227ccee.jpg"  alt="PHP" style="width:220px;height:172px;"/>
</a>
<span class="description">
PHP
</span>
</div>更新，以鼓励用户转移到PHP 5。</div>
<div class="para" label-module="para">2008年PHP 5成为了PHP唯一的有在开发的PHP版本。将来的PHP 5.3将会加入Late static binding和一些其他的功能强化。PHP 6 的开发也正在进行中，主要的改进有移除register_globals、magic quotes 和 Safe mode的功能。</div>
<div class="para" label-module="para">PHP最新稳定版本：5.4.30(2013.6.26)</div>
<div class="para" label-module="para">PHP最新发布的正式版本：5.5.14(2014.6.24)</div>
<div class="para" label-module="para">PHP最新测试版本：5.6.0 RC2(2014.6.03)</div>
<div class="para" label-module="para">2013年6月20日，PHP开发团队自豪地宣布推出PHP 5.5.0。此版本包含了大量的新功能和bug<a target=_blank href="/item/%E4%BF%AE%E5%A4%8D">修复</a>。需要开发者特别注意的一点是不再支持 Windows XP 和 2003 系统。</div>
<div class="para" label-module="para">2014年10月16日，PHP开发团队宣布PHP 5.6.2可用。四安全相关的错误是固定在这个版本，包括修复cve-2014-3668，cve-2014-3669和cve-2014-3670。所有的PHP 5.6鼓励用户升级到这个版本。</div><div class="anchor-list">

<h2 class="title-text"><span class="title-prefix">PHP</span>特性</h2>
</div>

PHP版本VC6与VC9、Thread Safe与None-Thread Safe等的区别</div>
<div class="para" label-module="para">PHP的大版本主要分三支：PHP4/PHP5/PHP6</div>
<div class="para" label-module="para">其中，PHP4由于太古老、对QQ支持不力已基本被淘汰，请无视PHP4。</div>
<div class="para" label-module="para">PHP6由于基本没有生产线上的应用，还基本只是一款概念产品，很多功能已在PHP5.3.3上实现，所以也不详述，请无视PHP6。</div>
<div class="para" label-module="para">PHP5的版本主要分四支：PHP5.2之前的版本、PHP5.2.X、PHP5.3和日前发布的PHP5.4。</div><div class="anchor-list">
<a name="7_1" class="lemma-anchor para-title" ></a>
<a name="sub5828265_7_1" class="lemma-anchor " ></a>
<a name="选择版本" class="lemma-anchor " ></a>
<a name="7-1" class="lemma-anchor " ></a>
</div><div class="para-title level-3" label-module="para-title">
<h3 class="title-text"><span class="title-prefix">PHP</span>选择版本</h3>
</div>
<div class="para" label-module="para">PHP5.2之前的版本不值得考虑，因为某些功能缺陷或者BUG。PHP5.4还处于Beta试用的版本号，非稳定版本，请无视PHP5.4。</div>
<div class="para" label-module="para">主流PHP程序对PHP5.2.X的兼容性最好，而每次版本号的升级带来的都是安全性和稳定性的改善，所以宜挑选最新的版本。目前PHP5.2系列最新的是PHP5.2.17。</div>
<div class="para" label-module="para">而如果产品是自己开发自己使用，PHP5.3在某些方面更具优势，在稳定性上更胜一筹，增加了很多PHP5.2所不具有的功能，比如内置php-fpm、更完善的垃圾回收算法、命名空间的引入、sqlite3的支持等等，是部署项目值得考虑的版本，强烈推荐PHP5.3.3。</div>
<div class="para" label-module="para">除了版本号的不同，同一版本号的PHP版本也有区别，并且在选择PHP扩展的时候需要注意。</div>
<ul class="custom_dot para-list list-paddingleft-1"><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">install版：可执行的MSI格式安装包。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">ZIP版：解压即可用。和install版无区别。建议选择ZIP版。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">DEBUG版：最终用户无需关心。</div>
</li></ul><div class="anchor-list">
<a name="7_2" class="lemma-anchor para-title" ></a>
<a name="sub5828265_7_2" class="lemma-anchor " ></a>
<a name="VC6与VC9" class="lemma-anchor " ></a>
<a name="7-2" class="lemma-anchor " ></a>
</div><div class="para-title level-3" label-module="para-title">
<h3 class="title-text"><span class="title-prefix">PHP</span>VC6与VC9</h3>
</div>
<div class="para" label-module="para">如果你在apache1或者apache2下使用PHP，你应该选择VC6的版本</div>
<div class="para" label-module="para">如果你在IIS下使用PHP应该选择VC9的版本，VC6的版本使用visual studio6编译，VC9使用Visual Studio 2008编译，并且改进了性能和稳定性。VC9版本的PHP需要你安装Microsoft 2008 C++ Runtime。不要在apache下使用VC9的版本。</div><div class="anchor-list">
<a name="7_3" class="lemma-anchor para-title" ></a>
<a name="sub5828265_7_3" class="lemma-anchor " ></a>
<a name="TS和NTS" class="lemma-anchor " ></a>
<a name="7-3" class="lemma-anchor " ></a>
</div><div class="para-title level-3" label-module="para-title">
<h3 class="title-text"><span class="title-prefix">PHP</span>TS和NTS</h3>
</div>
<div class="para" label-module="para">TS指Thread Safety，即线程安全，一般在IIS以ISAPI方式加载的时候选择这个版本。</div>
<div class="para" label-module="para">NTS即None-Thread Safe，一般以fast cgi方式运行的时候选择这个版本，具有更好的性能。</div>
<div class="para" label-module="para">从2000年10月20日发布的第一个Windows版的PHP3.0.17开始的都是线程安全的版本，这是由于与Linux/Unix系统是采用多进程的工作方式不同的是Windows系统是采用多线程的工作方式。如果在IIS下以CGI方式运行PHP会非常慢，这是由于CGI模式是建立在多进程的基础之上的，而非多线程。一般我们会把PHP配置成以ISAPI的方式来运行，ISAPI是多线程的方式，这样就快多了。但存在一个问题，很多常用的PHP扩展是以Linux/Unix的多进程思想来开发的，这些扩展在ISAPI的方式运行时就会出错搞垮IIS。因此在IIS下CGI模式才是 PHP运行的最安全方式，但CGI模式对于每个HTTP请求都需要重新加载和卸载整个PHP环境，其消耗是巨大的。</div>
<div class="para" label-module="para">为了兼顾IIS下PHP的效率和安全，微软给出了FastCGI的解决方案。FastCGI可以让PHP的进程重复利用而不是每一个新的请求就重开一个进程。同时FastCGI也可以允许几个进程同时执行。这样既解决了CGI进程模式消耗太大的问题，又利用上了CGI进程模式不存在线程安全问题的优势。</div>
<div class="para" label-module="para">因此，如果是使用ISAPI的方式来运行PHP就必须用Thread Safe(线程安全)的版本；而用FastCGI模式运行PHP的话就没有必要用线程安全检查了，用None Thread Safe(NTS，非线程安全)的版本能够更好的提高效率。</div><div class="anchor-list">
<a name="7_4" class="lemma-anchor para-title" ></a>
<a name="sub5828265_7_4" class="lemma-anchor " ></a>
<a name="查看版本" class="lemma-anchor " ></a>
<a name="7-4" class="lemma-anchor " ></a>
</div><div class="para-title level-3" label-module="para-title">
<h3 class="title-text"><span class="title-prefix">PHP</span>查看版本</h3>
</div>
<div class="para" label-module="para">一个很简单的办法就是phpinfo();</div>
<div class="para" label-module="para">Thread Safety disabled是NTS，enabled是TS</div>
<div class="para" label-module="para">Configure Command看到VC98字样的是VC6，Compiler标明 MSVC9 (Visual C++ 2008) 的是VC9</div>
<div class="para" label-module="para">在WIN7下：IIS7+NTS+FastCGI+vc9 是最佳搭档或者apache+fastcgi+nts+vc6。</div>
<div class="para" label-module="para">在WINXP下：Apache+TS+Apache module +vc6最合适的搭档。</div><div class="anchor-list">

<h3 class="title-text"><span class="title-prefix">PHP</span>简介</h3>
</div>
<div class="para" label-module="para">PHP4.0整个<a target=_blank href="/item/%E8%84%9A%E6%9C%AC%E7%A8%8B%E5%BA%8F">脚本程序</a>的核心大幅更动，让程序的执行速度，满足更快的要求。在最佳化之后的效率，已较传统<a target=_blank href="/item/CGI">CGI</a>或者<a target=_blank href="/item/ASP">ASP</a>等程序有更好的表现。而且还有更强的新功能、更丰富的函数库。无论您接不接受，PHP 都将在 Web CGI 的领域上，掀起巅覆性的革命。对于一位专业的Web Master 而言，它将也是必修课程之一。</div>
<div class="para" label-module="para">PHP 4.0是更有效的，更可靠的动态Web页开发工具，在大多数情况运行比 PHP 3.0要快，其脚本描述更强大并且更复杂，最显著的特征是速率比的增加。PHP4.0这些优异的性能是PHP<a target=_blank href="/item/%E8%84%9A%E6%9C%AC%E5%BC%95%E6%93%8E">脚本引擎</a>重新设计产生的结果：引擎由AndiGutmans和Zeev Suraski从底层全面重写。PHP4.0 脚本引擎 ——<a target=_blank href="/item/Zend">Zend</a>引擎，使用了一种更有效的编译——执行方式,而不是PHP 3.0 采用的执行 ——当解析时<a target=_blank href="/item/%E6%A8%A1%E5%9E%8B">模型</a>。</div><div class="anchor-list">

</div><div class="para-title level-3" label-module="para-title">
<h3 class="title-text"><span class="title-prefix">PHP</span>优越性</h3>
</div>
<div class="para" label-module="para">PHP4在3.0版的基础上增加或增强了许多有用的特征，主要如下：</div>
<div class="para" label-module="para">（1）别名：在PHP4中，可以利用引用为变量赋值，这给<a target=_blank href="/item/%E7%BC%96%E7%A8%8B">编程</a>带来了很大的灵活性。</div>
<div class="para" label-module="para">（2）扩充了<a target=_blank href="/item/API">API</a>模块：PHP 4.0 为扩展的 API 模块的提供了扩展PHP接口模块，它比旧的 API 版本显著地快。PHP 模块已有的及最常用的接口多数被转换到使用这个扩展的接口。</div>
<div class="para" label-module="para">（3）自动资源释放：PHP4增加了引用计数功能，这种新技术的引入使PHP4具有了自动<a target=_blank href="/item/%E5%86%85%E5%AD%98%E7%AE%A1%E7%90%86">内存管理</a>功能，减轻了开发人员的负担。</div>
<div class="para" label-module="para">（4）<a target=_blank href="/item/%E5%B8%83%E5%B0%94">布尔</a>类型：PHP 4.0 支持布尔类型。</div>
<div class="para" label-module="para">（5）进程生成：在 UNIX 环境下的 PHP 4.0 提供了一个很智能和通用的生成进程，使用了一种名为基于automake/libtool的系统生成技术。</div>
<div class="para" label-module="para">（6）COM/DCOM 支持：PHP 4.0 提供 COM/DCOM 支持 ( 仅用于Windows 环境 ) 可以无缝地存取和访问 COM 对象。</div>
<div class="para" label-module="para">（7）与PHP 3.0 兼容性很好：PHP 4.0 是与 PHP 3.0 代码向后<a target=_blank href="/item/%E5%85%BC%E5%AE%B9%E6%80%A7">兼容性</a>接近100%。由于 PHP 4 的改进的体系结构，两者有一些细微的差别，但是大多数人将可能永远不可能遇上这种情况。</div>
<div class="para" label-module="para">（8）配置：PHP4重新设计和增强了PHP.ini文件，这使得用PHP.ini来<a target=_blank href="/item/%E9%85%8D%E7%BD%AE">配置</a>PHP显得极为容易，这个文件可以在运行时被Apache(unix系统）或由Windows 注册（Windows 环境）。</div>
<div class="para" label-module="para">（9）加密支持：PHP4实现了完整的加密，这些加密功能是一个完整的mycrypt库，并且 PHP 4.0 支持哈希函数。Blowfish，TripleDES,MD5，并且SHA1 也是可使用的一些加密算法。</div>
<div class="para" label-module="para">（10）类型检查：PHP 4.0 支持同一操作符用于评类型检查：===(3 等号<a target=_blank href="/item/%E8%BF%90%E7%AE%97%E7%AC%A6">运算符</a>，恒等计算符），为在两个值和其类型之间作检查。例如，&#39;3&#39; ===3 将视为假 ( 类型是不同的），而 3 ==3 ( 相等判断，不比较类型 ) 将视为真。</div>
<div class="para" label-module="para">（11）FTP 支持：PHP 4.0 支持 FTP。通常，你会为通过一个调制解调器连接下载一个大文件提供一个接口。然而，如果你确实有需要，可以使用PHP。</div>
<div class="para" label-module="para">（12）PHP4新增函数或功能增强函数：PHP 4.0 新增了许多<a target=_blank href="/item/%E5%87%BD%E6%95%B0">函数</a>，同时也将许多现有的函数功能进行了增强，以下是一些例子。<a target=_blank href="/item/array_count_values">array_count_values</a>() eval() foreach() include() ob_end_clean() ob_end_flush() ob_get_contents() ob_start()<a target=_blank href="/item/strip_tags">strip_tags</a>() unset（）。</div>
<div class="para" label-module="para">（13）here打印：PHP 4.0 的Here打印是与Perl类似的，尽管完全不相同。Here是打印大容量文章的一个有用的方法，例如在<a target=_blank href="/item/HTML%E6%96%87%E4%BB%B6">HTML文件</a>中，不会漏掉任何一个字符，例如目录标记。</div>
<div class="para" label-module="para">（14）HTTP Session fallback 系统：为 HTTP Session管理的一个 fallback 系统在 PHP 4.0被实现。缺省情况下，Session标识符由cookies存储。如果没有cookies支持或一项cookies任务失败，Session标识符自动被创建并在 URL 的查询字符串中被携带。</div>
<div class="para" label-module="para">（15）ISAPI 支持：PHP 4.0 能作为一个<a target=_blank href="/item/%E4%B8%AA%E6%80%A7%E5%8C%96">个性化</a>的 ISAPI 模块作为<a target=_blank href="/item/IIS">IIS</a>插件。这比 PHP 3.0 更有效，它作为一个外部的程序来运行。</div>
<div class="para" label-module="para">（16）内存：PHP 4.0 能更有效的使用<a target=_blank href="/item/%E5%86%85%E5%AD%98">内存</a>，导致较少的内存占用消耗，这主要归功于引用计数技术的实现。⒄其他类<a target=_blank href="/item/%E6%88%90%E5%91%98%E5%87%BD%E6%95%B0">成员函数</a>：在 PHP 4.0 你能在成员函数本身的作用域或全局范围内调用其他类的成员函数。例如，你能用一个子<a target=_blank href="/item/%E5%87%BD%E6%95%B0%E8%A6%86%E7%9B%96">函数覆盖</a>父函数，并在子函数中调用父函数。</div>
<div class="para" label-module="para">（17）多维<a target=_blank href="/item/%E6%95%B0%E7%BB%84">数组</a>：在 PHP 4.0 ，利用GET，POST,Cookies的进行的数据传输支持多维数组。</div>
<div class="para" label-module="para">（18）个性化的 HTTP<a target=_blank href="/item/Session">Session</a>支持：HTTP Session处理，包括 fallback<a target=_blank href="/item/%E7%B3%BB%E7%BB%9F%E7%AE%A1%E7%90%86">系统管理</a>，在 PHP 4.0被它的新库函数实现。在版本 3.0 中处理Session要求使用 PHPLIB 和<a target=_blank href="/item/%E7%AC%AC%E4%B8%89%E6%96%B9">第三方</a>的库函数，它比把Session直接地由 PHP 支持慢了许多。</div>
<div class="para" label-module="para">（19）个性化的 Java 支持：PHP 4.0 支持和java的交互。这种个性化的Java 支持为PHP 在 Java 对象上创建和使用方法提供一个简单并且有效的工具。</div>
<div class="para" label-module="para">（20）对象和数嵌套组：PHP 4.0 实现了功能更加强大的对象，移去了 PHP 3.0存在的种种句法限制。对象能在数组以内被嵌套并且反过来也如此，可以根据你的需要实现嵌套。</div>
<div class="para" label-module="para">（21）面向对象的<a target=_blank href="/item/%E7%BC%96%E7%A8%8B">编程</a>：PHP 4.0 为面向对象的编程和构造类及对象提供扩展的功能和新特征。PHP4实现了对象重载，引用技术等新技术。</div>
<div class="para" label-module="para">（22）对象<a target=_blank href="/item/%E9%87%8D%E8%BD%BD">重载</a>支持：对象重载语法允许第三方的基于面向对象的类库使用 PHP4 的面向对象的特征存取他们自身的功能。使用这个特征的一个 COM 模块已经被实现了。</div>
<div class="para" label-module="para">（23）输出缓冲支持：PHP 提供了一个输出缓冲函数集合。输出缓冲支持允许你写<a target=_blank href="/item/%E5%8C%85%E8%A3%B9%E5%87%BD%E6%95%B0">包裹函数</a>功能压缩<a target=_blank href="/item/%E7%BC%93%E5%86%B2%E5%8C%BA">缓冲区</a>。在 PHP4 的输出缓冲支持允许 HTML 头信息存放，无论 HTML的正文是否输出。头信息（ (header(),content type,and cookies ) 不采用缓冲。</div>
<div class="para" label-module="para">（24）增加了<a target=_blank href="/item/PCRE">PCRE</a>库：PHP 4.0 包括一个 Perl 兼容的<a target=_blank href="/item/%E6%AD%A3%E5%88%99%E8%A1%A8%E8%BE%BE%E5%BC%8F">正则表达式</a>(PCRE ) 库，和正常regex库一起与 PHP 绑定。split 和replace PCRE 功能被支持。PCRE 和 Perl 正规表达式之间有一些细微差别。</div>
<div class="para" label-module="para">（25)PHP.ini 文件：PHP.ini文件在 PHP4.0 被重新设计，使用的 PHP 的配置PHP.ini是更容易并且更有效的。全部文件能被Apache 在运行时间操作 ( 在 Apache环境 下 ) 或由 Windows 注册表 ( 在 Windows 下面 ）。被加入PHP.ini文件的配置指令自动地在所有相关的模块中被支持。</div>
<div class="para" label-module="para">（26）引用计数：PHP 4.0 为系统中的每个数值提供了引用计数，包括资源。一旦一个资源不再被任何变量引用，它自动地被释放以节省内存资源。利用这个特征的最明显的例子一个内置SQL查询的循环语句。在PHP 3.0中 ，每次递归另外的 SQL 结果集合重复申请内存，直到脚本执行完毕，这些结果集合占用的内存才被释放。</div>
<div class="para" label-module="para">（27）支持引用：通过引用可以改变一个变量的值。</div>
<div class="para" label-module="para">（28）函数的运行时绑定：PHP 4.0 的运行时间绑定功能允许你在他们被声明以前调用，无论声明是否在代码以后或是在运行时间。</div>
<div class="para" label-module="para">（29）类的运行时信息：PHP 4.0 支持在运行时刻存取下列类信息：一个对象的类名，一个对象的父类的类名字，以及对象函数所在的名字。</div>
<div class="para" label-module="para">（30）<a target=_blank href="/item/%E6%9C%8D%E5%8A%A1%E5%99%A8">服务器</a>抽象层：为支持Web服务器提供了增强型 SAPI ( 服务器 API ) 接口，是 PHP 4.0 不可分的一部分。这个服务器抽象层，提供了通用的<a target=_blank href="/item/WEB%E6%9C%8D%E5%8A%A1%E5%99%A8">WEB服务器</a>接口支持，支持<a target=_blank href="/item/%E5%A4%9A%E7%BA%BF%E7%A8%8B">多线程</a>WEB服务器，为大多数的WEB服务器提供透明的支持，这些服务器包括 Apache,IIS ( ISAPI ），以及 AOL 服务器。</div>
<div class="para" label-module="para">（31）语法的点亮显示：PHP 4.0 语法的点亮显示允许开发者看见源代码而不是脚本，这个功能比PHP 3.0中的更有效。它跑得更快，更执行得更好，并且产生更紧凑的HTML代码。</div>
<div class="para" label-module="para">（32）由引用改变<a target=_blank href="/item/%E5%8F%98%E9%87%8F">变量</a>的值：PHP 4.0 由引用支持可变的赋值，“关联”的2个变量之中个的任何一个的值被改变，另外的变量的值同样被改变，这类似与C中的指针类型。</div>
<div class="para" label-module="para">（33）在引用字符串中的变量引用：PHP 4.0 增强了在引用字符串中的变量引用。</div><div class="anchor-list">

<h2 class="title-text"><span class="title-prefix">PHP</span>PHP5</h2>
</div>
<div class="para" label-module="para">PHP5在长时间的开发及多个预发布版本后，2004年7月13日，PHP5.0发布。该版本以<a target=_blank href="/item/Zend%E5%BC%95%E6%93%8E">Zend引擎</a>Ⅱ为引擎，并且加入了新功能如PHP Data Objects（<a target=_blank href="/item/PDO">PDO</a>）。PHP5.0版本强化更多的功能。首先，完全实现面向对象，提供名为<b>PHP兼容模式</b>的功能。其次是XML功能，PHP5.0版本支持可直观地访问XML数据、名为SimpleXML的XML处理用界面。同时还强化了<a target=_blank href="/item/XML">XML</a>Web服务支持，而且标准支持SOAP扩展模块。数据库方面，PHP新版本提供旨在访问MySQL的新界面——<b>MySQL</b>。除此前的界面外，还可以使用面向对象界面和预处理语句（Prepared Statement）等MySQL的新功能．另外，PHP5.0上还捆绑有小容量RDBMS-SQLite．</div><div class="anchor-list">

<a name="新对象模式" class="lemma-anchor " ></a>
<a name="10-1" class="lemma-anchor " ></a>
</div><div class="para-title level-3" label-module="para-title">
<h3 class="title-text"><span class="title-prefix">PHP</span>新对象模式</h3>
</div>
<div class="para" label-module="para">*<a target=_blank href="/item/%E6%9E%84%E9%80%A0%E5%87%BD%E6%95%B0">构造函数</a>和<a target=_blank href="/item/%E6%9E%90%E6%9E%84%E5%87%BD%E6%95%B0">析构函数</a> * 对象的引用 * 对象的克隆 * 对象中的私有、公共及受保护模式 * 接口 (Interfaces)</div>
<div class="para" label-module="para">* 抽象类 * __call * __set 和 __get * 静态成员</div>
<div class="para" label-module="para"><b>构造函数和析构函数</b></div>
<div class="para" label-module="para">在 PHP4 中，当函数与对象同名时，这个函数将成为该对象的构造函数，并且在 PHP4 中没有析构函数的概念。</div>
<div class="para" label-module="para">在 PHP5 中，构造函数被统一命名为 __construct，并且引入了析构函数的概念，被统一命名为 __destruct。</div>
<div class="para" label-module="para">对象的引用</div>
<div class="para" label-module="para">在PHP4中，传递变量给一个函数或方法，实际是把这个变量做了一次复制，也就意味着你传给函数或方法的是这个变量的一个副本，除非你使用了引用符号“&amp;；” 来声明是要做一个引用，而不是一个 Copy。在 PHP5中，对象总是以引用的形式存在的，对象中的赋值操作同样也都是一个引用操作。</div>
<div class="para" label-module="para">对象的克隆</div>
<div class="para" label-module="para">当一个对象始终以引用的形式来被调用时，如果我想得到该对象的一个副本，该怎么办呢？PHP5 提供了一个新的功能，就是对象的克隆，语法为 __clone。</div>
<div class="para" label-module="para"><b>抽象</b><a target=_blank href="/item/%E7%B1%BB">类</a></div>
<div class="para" label-module="para">抽象类不能被实例化。</div>
<div class="para" label-module="para">抽象类与其它类一样，允许定义变量及方法。</div>
<div class="para" label-module="para">抽象类同样可以定义一个抽象的方法，抽象类的方法不会被执行，不过将有可能会在其派生类中执行。</div>
<div class="para" label-module="para"><b>__call</b></div>
<div class="para" label-module="para">PHP5 的对象新增了一个专用方法 __call（），这个方法用来监视一个对象中的其它方法。如果你试着调用一个对象中不存在的方法，__call 方法将会被自动调用。</div>
<div class="para" label-module="para"><b>__set 和 __get</b></div>
<div class="para" label-module="para">这是一个很棒的方法，__set 和 __get 方法可以用来捕获一个对象中不存在的变量和方法。</div>
<div class="para" label-module="para"><b>类型指示</b></div>
<div class="para" label-module="para">在 PHP5 中，你可以在对象的方法中指明其参数必须为另一个对象的实例。</div>
<div class="para" label-module="para">静态成员</div>
<div class="para" label-module="para">静态成员和静态方法在面象对象编程的术语中被称作 “类方法（class methods）” 和 “类变量（class variables）”。</div>
<div class="para" label-module="para">“类方法” 在一个对象没有实例化前允许被调用。同样，“类变量” 在一个对象没有实例化前可以被独立操作控制（不需要用一个对象的方法来控制）。</div><div class="anchor-list">

<h3 class="title-text"><span class="title-prefix">PHP</span>异常处理</h3>
</div>
<div class="para" label-module="para">异常处理是公认的处理<a target=_blank href="/item/%E7%A8%8B%E5%BA%8F%E9%94%99%E8%AF%AF">程序错误</a>的理想方法，在 <a target=_blank href="/item/Java/85979" data-lemmaid="85979">Java</a>及 C 中都有这个概念，我们欣喜的看到，在 PHP5 已经加入了这方面的应用。你可以尝试使用 “try” 和 “catch” 来控制程序的错误。当有错误发生的时候，代码会把错误交给 “catch” 子句来处理，在 “catch” 子句中，你需要指明要把错误交给某个对象处理，这样做可以使代码结构看起来更清晰，因为我们可以把所有的错误信息交给一个对象来处理。</div>
<div class="para" label-module="para"><b>自定义错误处理</b></div>
<div class="para" label-module="para">你可以很方便的用自定义的处理错误的代码来控制你的程序中的意外。你仅仅需要从异常类中派生出一个自己的错误控制类，在你自己的错误控制类中，你需要有一个构造函数和一个 getMessage 方法。</div><div class="anchor-list">

<h3 class="title-text"><span class="title-prefix">PHP</span>命名空间</h3>
</div>
<div class="para" label-module="para">名称空间对类的分组或函数分组很有用。它可以把一些相关的类或函数给组合到一起，方便以后调用。</div>
<div class="para" label-module="para">例：名称空间</div>
<pre class="brush: php">&lt;?php
namespacemy\name;//参考&quot;定义命名空间&quot;小节
classMyClass{}
functionmyfunction(){}
constMYCONST=1;
$a=newMyClass;
$c=new\my\name\MyClass;//参考&quot;全局空间&quot;小节
$a=strlen(&#39;hi&#39;);//参考&quot;使用命名空间：后备全局函数/常量&quot;小节
$d=namespace\MYCONST;//参考&quot;namespace操作符和__NAMESPACE__常量”小节
$d=__NAMESPACE__.&#39;\MYCONST&#39;;
echoconstant($d);//参考&quot;命名空间和动态语言特征&quot;小节
?&gt;
</pre><div class="para" label-module="para">注意你需要在何种情况下使用名称空间，在实际运用中，你可能会需要声明两个或多个名称一样的对象来做不同的事情，那么你就可以把他们分别放到不同的名称空间中去（但接口是要相同的）。</div><div class="anchor-list">

<div class="para" label-module="para">Linux安装Zend Guard Loader支持的过程:</div>
<div class="para" label-module="para">操作系统为CentOS5.5，PHP版本为5.3.8（CentOS5.5中的PHP默认版本较低，如果要升级到PHP最新版，可以使用remi的report源进行升级）。</div>
<div class="para" label-module="para">Windows 下面的 Zend Guard Loader 不支持php5.3.8 VC9 x86 Thread Safe而，php5.3.8 VC9 x86 Non Thread Safe 又不支持 apache。所以安装的话，就需要安装在iis或者和nginx搭配。</div>
<div class="para" label-module="para">1、下载最新的Linux操作系统系下的Zend Guard Loader。</div>

</pre><div class="para" label-module="para">注意路径一定要写上，刚开始没写路径，老是加载不上ZendGuardLoader .so，也可以直接将上面的配置写入/etc/php.ini文件中，效果一样。</div>
<div class="para" label-module="para">3、重启httpd服务：service httpd restart。</div>
<div class="para" label-module="para">4.输出&lt;?php phpinfo(); ?&gt;</div>

<a name="版本6设想" class="lemma-anchor " ></a>
</div><div class="para-title level-2" label-module="para-title">
<h2 class="title-text"><span class="title-prefix">PHP</span>版本6设想</h2>
</div>
<div class="para" label-module="para">1、<b>支持<a target=_blank href="/item/Unicode">Unicode</a></b>。Unicode是有其必然，虽然Unicode占用较多的空间，但Unicode带来的便利性，远超过占用空间的缺点，尤其在国际化的今天，硬件设备越来越强大，<a target=_blank href="/item/%E7%BD%91%E9%80%9F">网速</a>也大幅度的提升，这么一点小小的缺点是可以忽略的。另外一点，PHP也可以在.ini文件中设定能不能开启支持Unicode，决定权在你自己，这是一个不错的点子，关掉Unicode的支持，PHP的性能并不会有大幅度的提升，主要的影响在于需要引用字符串的函数。</div>
<div class="para" label-module="para">2、<b>Register Globals 将被移除</b>这是一个主要的决定，老的PHP使用者会觉得Register Globals满方便的，但是却忽略了Register Globals会带来程序上安全性的隐患，大多数的主机上此项功能是关闭的，印象中从PHP4.3.x版开始，此项默认配置值即是关闭状态，PHP6正式移除Register Globals也代表着如果程序是PHP3时代的产物，将完全不能运用，除了改写用途外，别无他法。</div>
<div class="para" label-module="para">3、<b>Magic Quotes 将取消</b>Magic Quotes主要是自动转义须要转义的字符，此项功能移除也符合大多数PHP开发者的心声。</div>
<div class="para" label-module="para">4、<b>Safe Mode 取消</b>。老实说，这个模式不知道哪里不好，取消就取消吧，反正也用不到。</div>
<div class="para" label-module="para">5、<b>’var’ 别名为 ‘public’</b>，在类中的var声明变成public的别名，相信是为了兼容PHP5而作的决定，PHP6现在也可以称作为OO语言了。</div>
<div class="para" label-module="para">6、<b>通过引用返回将出错</b>未来通过引用返回编译器将会报错 例如$a =&amp; new b()、function &amp;c()，OO语言默认就是引用，所以不须要再运用 &amp;了。</div>
<div class="para" label-module="para">7、<b>zend.ze1 compatbility mode 将被移去</b>（Zend.ze1相容模式将被移去），PHP5是为兼容旧有PHP4，所以在.ini中可选择是否开启相容模式，原由在于PHP5运用的是第二代分析引擎，但是相容模式并不是百分之百能分析 PHP4语法，所以旧时代的产物，移除。</div>
<div class="para" label-module="para">8、<b>Freetype 1 and GD 1 support 将不见</b>这两个Libs存在了很久，php6将不再支持，况且GD1早已被现在的GD2取代了。</div>
<div class="para" label-module="para">9、<b>dl() 被移到 SAPI 中dl()主要是让设计师加载extension Libs</b>，被移到 SAPI 中。</div>
<div class="para" label-module="para">10、<b>Register Long Array 去除 从PHP5起默认是关闭</b>，在PHP6中正式移除。</div>
<div class="para" label-module="para">11、<b>一些Extension的变更例如 XMLReader 和 XMLWriter 将不再是以Extension的方式出现</b>，他们将被移入到PHP的核心之中，并且默认是开启，<a target=_blank href="/item/ereg">ereg</a>extension将被放入PECL，代表着它将被移出PHP核心，这也是为了让路给新的<a target=_blank href="/item/%E6%AD%A3%E5%88%99%E8%A1%A8%E8%BE%BE%E5%BC%8F">正则表达式</a>extension，此外，Fileinfo extension 也将被导入PHP的核心之中。</div>
<div class="para" label-module="para">12、<b>APC将被导入核心</b>这是一个提高PHP性能的功能，它将被放入PHP核心中，并且可以选择是否启用APC。</div>
<div class="para" label-module="para">13、<b>告别ASP风格的起始标签</b>原来是为了取悦ASP开发者转向运用 PHP，现今已经不再须要这种做法了。</div>
<div class="para" label-module="para">最后，别期望PHP6的性能可以彻底超过PHP5，有可能PHP6的执行效率会比PHP5还要来得慢，但是可以预期的是，PHP开发小组将会努力的完善PHP6，超越PHP5。</div><div class="anchor-list">

<div class="para" label-module="para">不同的作者之间说法可能不一样，但是一个<a target=_blank href="/item/OOP">OOP</a>语言必须有以下几方面：</div>
<div class="para" label-module="para">1.<a target=_blank href="/item/%E6%8A%BD%E8%B1%A1%E6%95%B0%E6%8D%AE%E7%B1%BB%E5%9E%8B">抽象数据类型</a>和信息封装</div>
<div class="para" label-module="para">2.<a target=_blank href="/item/%E7%BB%A7%E6%89%BF">继承</a></div>
<div class="para" label-module="para">3.<a target=_blank href="/item/%E5%A4%9A%E6%80%81">多态</a></div>
<div class="para" label-module="para">在PHP中是通过类来完成封装的：</div>
<pre class="brush: php">//在OOP类中，通常采用大双驼峰命名法，每个单词的首字母都大写
classSomething{
//作用域修饰符：public公共的；private私有的；protected受保护的；
//属性的名称一般用全小写
private$x=null;//在编程建议中，内部使用的属性应该给私有修饰符，然后通过方法取值赋值

//方法的名称一般用小驼峰命名法，第一个单词全小写，剩下的单词首字母大写
//因为PHP不会自动为变量使用$this所以必须主动加上$this伪变量来指向操作的对象
publicfunctionsetX($v){
$this-&gt;x=$v;
}
publicfunctiongetX(){
return$this-&gt;x;
}
}
</pre><div class="para" label-module="para">当然你可以按自己的喜好进行定义，但最好保持一种标准，这样会更有效。数据成员在类中使用&quot;var&quot;声明来定义，在给数据成员赋值之前，它们是没有类型的。一个数据成员可以是一个整数，一个数组，一个相关数组（associative array）或者是一个对象。方法在类中被定义成函数形式，在方法中访问类<a target=_blank href="/item/%E6%88%90%E5%91%98%E5%8F%98%E9%87%8F">成员变量</a>时，你应该使用$this-&gt;name，否则对一个方法来说，它只能是<a target=_blank href="/item/%E5%B1%80%E9%83%A8%E5%8F%98%E9%87%8F">局部变量</a>。</div>
<div class="para" label-module="para">使用new操作符来创建一个对象：</div>
<pre class="brush: php">$obj=newSomething;
</pre><div class="para" label-module="para">然后你可以使用成员函数通过：</div>
<pre class="brush: php">$obj-&gt;setX(5)；
$see=$obj-&gt;getX();
echo$see;</pre><div class="para" label-module="para">在这个例子中，setX成员函数将5赋值给对象的成员变量x（不是类的），然后getX返回它的值5。可以象：$obj-&gt;x=6那样通过类引用方式来存取数据成员，这不是一个很好的OOP习惯。我强烈建议通过方法来存取成员变量。如果你把成员变量看成是不可处理的，并且只通过对象句柄来使用方法，你将是一个好的OOP程序员。不幸的是，PHP不支持声明私有成员变量，所以不良代码在PHP中也是允许的。继承在PHP中很容易实现，只要使用extends关键字。</div>
<pre class="brush: php">classAnotherextendsSomething{
private$y;

publicfunctionsetY($v){
$this-&gt;y=$v;
}
functiongetY(){
return$this-&gt;y;
}
}
</pre><div class="para" label-module="para">&quot;Another&quot;类的对象拥有了父类（Something）的全部的<a target=_blank href="/item/%E6%95%B0%E6%8D%AE/33305" data-lemmaid="33305">数据</a>成员及方法，而且还加上了自己的数据成员和方法。</div>
<div class="para" label-module="para">你可以使用</div>
<pre class="brush: php">$obj2=newAnother;
$obj2-&gt;setY(5);
echo$obj2-&gt;getY();
</pre><div class="para" label-module="para">PHP只支持单<a target=_blank href="/item/%E7%BB%A7%E6%89%BF">继承</a>，所以你不能从两个或两个以上类派生出新的类来。你可以在派生类中重定义一个方法，如果我们在&quot;Another&quot;类中重定义了getX方法（方法重写），我们就不能使 用&quot;Something&quot;中的getX方法了。如果你在派生类中声明了一个与基派同名的数据成员，那么当你处理它时， 它将“隐藏”基类的数据成员。</div>
<div class="para" label-module="para">你可以在你的类中定义构造<a target=_blank href="/item/%E5%87%BD%E6%95%B0">函数</a>。构造<a target=_blank href="/item/%E5%87%BD%E6%95%B0">函数</a>是一个与类名同名的方法，当你创建一个类的对象时会被调用，例如：</div>
<pre class="brush: php">classSomething{
private$x=null;

//新版本的构造函数放弃使用类名，而统一使用__construct()
publicfunction__construct($x){
$this-&gt;x=$x;
}
publicfunctionsetX($v){
$this-&gt;x=$v;
}
publicfunctiongetX(){
return$this-&gt;x;
}

//析构函数
publicfunction__destruct(){

}
}
</pre><div class="para" label-module="para">所以你可以创建一个对象，通过：</div>
<pre class="brush: php">$obj=newSomething(6)；
</pre><div class="para" label-module="para">构造函数会自动地把6赋值给数据变量x。构造函数和方法都是普通的<a target=_blank href="/item/PHP%E5%87%BD%E6%95%B0">PHP函数</a>(”__“两个下划线，魔术方法)，所以你可以使用<a target=_blank href="/item/%E7%BC%BA%E7%9C%81%E5%8F%82%E6%95%B0">缺省参数</a>。</div>
<pre class="brush: php">publicfunction__construct($x=&quot;3&quot;,$y=&quot;5&quot;){
}
</pre><div class="para" label-module="para">接着：</div>
<pre class="brush: php">$obj=newSomething();//x=3andy=5
$obj=newSomething(8);//x=8andy=5
$obj=newSomething(8,9);//x=8andy=9
</pre><div class="para" label-module="para">缺省参数使用C++的方式，所以你不能忽略Y的值，而给X一个缺省参数，参数是从左到右赋值的，如果传入的参数少于要求的参数时，其作的将使用缺省参数。</div>
<div class="para" label-module="para">当一个派生类的对象被创建时，只有它的构造函数被调用，父类的构造函数没被调用，如果你想调用基类的构造函数，你必须要在派生类的构造函数中用parent::__construct()调用。可以这样做是在派生类中所有父类的方法都是可用的。</div>
<pre class="brush: php">classAnotherextendsSomething{

publicfunction__construct(){
parent::__construct(5,6);//显示调用基类构造函数
}
}

</pre><div class="para" label-module="para">OOP的一个很好的机制是使用抽象类。抽象类是不能实例化，只能提供给派生类一个接口。设计者通常使用抽象类来强迫程序员从基类派生，这样可以确保新的类包含一些期待的功能。在PHP中没有标准的方法，但是：如果你需要这个特性，可以通过定义基类，并在它的构造函数后加上&quot;die&quot; 的调用，这样就可以保证基类是不可实例化的，在每一个方法（接口）后面加上&quot;die&quot; 语句，所以，如果一个程序员在派生类中没有覆盖方法，将引发一个错误。而且因为PHP 是无类型的，你可能需要确认一个对象是来自于你的基类的派生类，那么在基类中增加一个方法来实义类的身份（返回某种标识id），并且在你接收到一个对象参数时校验这个值。当然，如果一个不好的邪恶程序员在派生类中覆盖了这个方法，这种方法就不起作用了，不过一般问题多在<a target=_blank href="/item/%E6%87%92%E6%83%B0">懒惰</a>的程序员身上，而不是邪恶的程序员。</div>
<div class="para" label-module="para">当然，能够让基类对程序员无法看到是很好的，只要将接口打印出来做他们的工作就可以了。PHP 5 引入了析构函数的概念，这类似于其它面向对象的语言，如 C++。析构函数会在到某个对象的所有引用都被删除或者当对象被显式销毁时执行。<sup class="sup--normal" data-sup="3">

<div class="para" label-module="para"><a target=_blank href="/item/%E9%87%8D%E8%BD%BD">重载</a>（与覆盖不同）在PHP中<b>不支持，因为PHP是弱类型语言</b>。在OOP中，你可以重载一个方法来实现两个或重多的方法具有相同的名字，但是有不同数量或类型的参数（这要看语言）。PHP 是一种松散类型的语言，所以通过类型重载不起作用，然而通过参数的个数不同来重载也不起作用。</div>
<div class="para" label-module="para">有时在OOP中重载构造函数非常好，这样你可以通过不同的方法创建对象（变量函数）。在PHP中实现它的技巧是：</div>

<div class="para" label-module="para">多态</div>
<div class="para" label-module="para">多态是对象的一种能力，它可以在运行时刻根据传递的对象参数，决定调用哪一个对象的方法。例如，如果你有一个figure的类，它定义了一个draw的方法。并且派生了circle和rectangle 类，在派生类中你覆盖了draw方法，你可能还有一个函数，它希望使用一个参数x，并且可以调用$x-&gt;draw（）。如果你有多态性，调用哪个draw方法就依赖于你传递给这个函数的对象类型。</div>
<div class="para" label-module="para">多态性在象PHP这样的解释语言（想象一下一个C++编译器生成这样的代码，你应该调用哪一个方法？你也不知道你拥有的对象是什么类型的，好，这不是重点）是非常容易和自然的。所以PHP当然支持多态性。</div>
<pre class="brush: php">classCalc{
functionniceDrawing($x){//假设这是Board类的一个方法
$x-&gt;draw();
}
}

classCircle{

}

$board=newCalc;
$obj=newCircle(3,187);

</pre><div class="para" label-module="para">用PHP进行面向对象编程</div>
<div class="para" label-module="para">一些&quot;纯化论者（purists)&quot;可能会说PHP不是一个真正的面向对象的语言，这是事实。PHP 是一个混合型语言，你可以使用OOP，也可以使用传统的过程化编程。然而，对于大型项目，你可能想/需要在PHP 中使用纯的OOP去声明类，而且在你的项目只用对象和类。</div>
<div class="para" label-module="para">随着项目越来越大，使用OOP可能会有帮助，OOP代码很容易维护，容易理解和重用。这些就是软件工程的基础。在基于web的项目中应用这些概念就成为将来网站成功的关键。</div>
<div class="para" label-module="para">高级OOP技术</div>
<div class="para" label-module="para">在看过基本的OOP概念后，我就可以向你展示更高级的技术：</div>
<div class="para" label-module="para">序列化（Serializing)</div>
<div class="para" label-module="para">PHP不支持永久对象，在OOP中永久对象是可以在多个应用的引用中保持状态和功能的对象，这意味着拥有将对象保存到一个文件或数据库中的能力，而且可以在以后装入对象。这就是所谓的序列化机制。PHP 拥有序列化方法，它可以通过对象进行调用，序列化方法可以返回对象的字符串表示。然而，序列化只保存了对象的成员数据而不包括方法。</div>
<div class="para" label-module="para">在PHP4中，如果你将对象序列化到字符串$s中，然后释放对象，接着反序列化对象到$obj，你可以继续使用对象的方法！我不建议这样去做，因为（a）文档中没有保证这种行为在以后的版本中仍然可以使用。（b) 这个可能导致一种误解，在你把一个序列化后的版本保存到磁盘并退出脚本时。当以后运行这个脚本时，你不能期待着在反序列化一个对象时，对象的方法也会在那里，因为字符串表示根本就不包括方法。</div>
<div class="para" label-module="para">总而言之，PHP 进行序列化对于保存对象的成员变量非常有用。（你也可以将相关数组和数组序列化到一个文件中）。</div>
<div class="para" label-module="para">例子 :</div>
<pre class="brush: php">$obj=newClassfoo();
$str=serialize($obj);//保存$str到磁盘上
$obj2=unserialize($str);//几个月以后//从磁盘中装入str
</pre><div class="para" label-module="para">你恢复了成员数据，但是不包括方法（根据文档所说）。这导致了只能通过类似于使用$obj2-&gt;x来存取成员变量（你没有别的方法！）的唯一办法，所以不要在家里试它。</div>
<div class="para" label-module="para">有一些办法可以解决这个问题，我把它留着，因为对这篇简洁的文章来说，他们太不好。我会很高兴地欢迎在PHP的后续版本中有全序列化的特性。</div>
<div class="para" label-module="para">使用类进行数据存储PHP和OOP一件非常好的事情就是，你可以很容易地定义一个类来操作某件事情，并且无论何时你想用的时候都可以调用相应的类。假设你有一个<a target=_blank href="/item/HTML%E8%A1%A8%E5%8D%95">HTML表单</a>，用户可以通过选择产品ID号来选择一个产品。在数据库中有产品的信息，你想把产品显示出来，显示它的价格等等。你拥有不同类型的产品，并且同一个动作可能对不同的产品具有不同的意思。例如，显示一个声音可能意味着播放它，但是对于其它种类的产品可能意味着显示一个存在数据库中的图片。你可以使用OOP或PHP来减少编码并提高质量：</div>
<div class="para" label-module="para">定义一个产品的类，定义它应该有的方法（例如：显示），然后定义对每一种类型的产品的类，从产品类派后出来（SoundItem类，ViewableItem类，等等），覆盖在产品类中的方法，使它们按你的想法动作。</div>
<div class="para" label-module="para">根据数据库中每一种产品的类型（type）字段给类命名，一个典型的产品表可能有（id,type,price,description，等等字段）...然后在处理脚本中，你可以从数据库中取出type值，然后实例化一个名为type的对象：</div>
<pre class="brush: php">$obj=new$type();
$obj-&gt;action();
</pre><div class="para" label-module="para">这是PHP的一个非常好的特性，你可以不用考虑对象的类型，调用$obj的显示方法或其它的方法。使用这个技术，你不需要修改脚本去增加一个新类型的对象，只是增加一个处理它的类。</div>
<div class="para" label-module="para">这个功能很强大，只要定义方法，而不去考虑所有对象的类型，在不同的类中按不同的方法实现它们，然后在主脚本中对任意对象使用它们，没有if...else，也不需要两个程序员，只有高兴。</div>
<div class="para" label-module="para">你同意编程是容易的，维护是便宜的，可重用是真的吗？</div>
<div class="para" label-module="para">如果你管理一组程序员，分配工作就是很简单的了，每个人可能负责一个类型的对象和处理它的类。</div>
<div class="para" label-module="para">可以通过这个技术实现国际化，根据用户所选的语言字段应用相应的类就可以了，等等。</div>
<div class="para" label-module="para">拷贝和克隆</div>
<div class="para" label-module="para">当你创建一个$obj的对象时，你可以通过$obj2=$obj来拷贝对象，新的对象是$obj的一个拷贝（不是一个引用），所以它具有$obj在当时的状态。有时候，你不想这样，你只是想生成一个象obj类一样的一个新的对象，可以通过使用new语句来调用类的构造函数。在PHP中也可以通过序列化，和一个基类来实现，但所有的其它类都要从基类派生出来。</div>
<div class="para" label-module="para">进入危险区域</div>
<div class="para" label-module="para">当你序列化一个对象，你会得到某种格式的字符串，如果你感兴趣，你可以调究它，其中，字符串中有类的名字（太好了！），你可以把它取出来，象：</div>
<pre class="brush: php">$herring=serialize($obj);
$vec=explode(&#39;:&#39;,$herring);//以:为标识符把字符串拆分成一个数组
$nam=str_replace(&quot;\&quot;,&#39;&#39;,$vec[2]);
</pre><div class="para" label-module="para">所以假设你创建了一个&quot;Universe&quot;的类，并且强制所有的类都必须从universe扩展，你可以在universe 中定义一个clone的方法，如下：</div>
<pre class="brush: php">classUniverse{

//在新的PHP版本中克隆（__clone()）是一个魔术方法，不要和这个方法搞混了

functionclone(){

$herring=serialize($this);

$vec=explode(&#39;:&#39;,$herring);

$nam=str_replace(&quot;\&quot;&quot;,&#39;&#39;,$vec[2]);

$ret=new$nam;
return$ret;

}

}//然后

$obj=newSomething();//从Universe扩展

$other=$obj-&gt;clone();
</pre><div class="para" label-module="para">你所得到的是一个新的Something类的对象，它同使用new方法，调用构造函数创建出的对象一样。我不知道这个对你是否有用，但是Universe类可以知道派生类的名字是一个好的经验。想象是唯一的限制。</div>

</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">SimpleTemplate：一个可以创建和结构化网站的模板引擎。它可以解析和编译模板。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">bTemplate：短小但是快速的模板类，允许你把PHP逻辑代码从HTML修饰代码中分离。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">Savant：一个强大且轻量级的PEAR兼容模板系统。它是非编译型的，使用PHP语言本身做为它的模板语言。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">ETS - easy template system：可以使用完全相同数据重组模板的模板系统。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">EasyTemplatePHP：适用于你的站点的一个简单但是强大的模板系统。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">vlibTemplate：一个快速、全能的模板系统，它包含一个缓存和调试类。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">AvanTemplate：多字节安全的模板引擎，占用很少系统资源。它支持变量替换，内容块可以设置显示或隐藏</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">Grafx Software’s Fast Template：一个修改版本的Fast Template系统，它包括缓存功能，调试控制台以及沉默去除为赋值块。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">TemplatePower：一个快速、简单、功能强大的模板类。主要功能有嵌套的动态块支持，块/文件包含支持以及显示/隐藏未赋值的变量。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">TagTemplate：这个库的功能被设计来使用模板文件，同时允许你从HTML文件检索信息。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">htmltmpl: templating engine：一个适用于Python和PHP的模板引擎。它面向希望在项目中分离代码和设计的web应用开发人员。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">PHP Class for Parsing Dreamweaver templates：一个分析<a target=_blank href="/item/Dreamweaver%E6%A8%A1%E6%9D%BF">Dreamweaver模板</a>的简单类，被用于Gallery 2 和WordPress的自定义模块中。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">MiniTemplator (Template Engine)：针对HTML文件的一个紧凑型模板引擎。对于模板变量和块定义它具有简单的语法。其中块可以嵌套。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">Layout Solution：简化<a target=_blank href="/item/%E7%BD%91%E7%AB%99%E5%BC%80%E5%8F%91">网站开发</a>和维护。它拥有常用的变量和页面元素使你不需要重复做页面布局工作。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">Cached Fast Template：它已经纳入 FastTemplate ，允许你缓存模板文件，甚至可以在分离的块内容上缓存不同的规格。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">TinyButStrong：一个支持MySQL,Odbc,Sql-Server和ADODB的模板引擎。它包含7个方法和两个属性。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">Brian Lozier’s php based template engine：只有2K大小，非常快并且是<a target=_blank href="/item/%E9%9D%A2%E5%90%91%E5%AF%B9%E8%B1%A1%E8%AE%BE%E8%AE%A1">面向对象设计</a>。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">WACT：一个从设计中分离代码的<a target=_blank href="/item/%E6%A8%A1%E6%9D%BF%E5%BC%95%E6%93%8E">模板引擎</a>。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">PHPTAL：一个PHP下面的XML/XHTML模板库。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">Rong_View_Wudimei：Wudimei开发的国产框架Rong Framework的模板引擎，它类似于smarty，优点是速度快，缺点是模板标签较少，不过够用了。</div>
</li></ul><div class="para" label-module="para"><b>框架介绍thinkphp</b></div>
<div class="para" label-module="para"><a target=_blank href="/item/ThinkPHP">ThinkPHP</a>是一个免费开源的，快速、简单的面向对象的 轻量级PHP开发框架 ，创立于2006年初，遵循Apache2开源协议发布，是为了敏捷WEB应用开发和简化企业应用开发而诞生的。ThinkPHP从诞生以来一直秉承简洁实用的设计原则，在保持出色的性能和至简的代码的同时，也注重易用性。并且拥有众多的原创功能和特性，在社区团队的积极参与下，在易用性、扩展性和性能方面不断优化和改进，已经成长为国内最领先和最具影响力的WEB应用开发框架，众多的典型案例确保可以稳定用于商业以及门户级的开发。</div>
<div class="para" label-module="para">PHP认证级别</div>
<div class="para" label-module="para">PHP课程由初级（IFE）、中级（IPE）和高级（IAE）三个部分。</div>
<ul class="custom_dot para-list list-paddingleft-1"><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">IFE即Index Front-end Engineer的缩写，指数前端工程师的意思。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">IPE即 Index PHP Engineer 的缩写，意思是指数PHP工程师。</div>
</li><li class="list-dot list-dot-paddingleft"><div class="para" label-module="para">IAE即 Index architecture/advanced engineer 的缩写，意思是：指数高级/架构工程师。</div>
</li></ul><div class="para" label-module="para">PHP安全</div>
<div class="para" label-module="para">PHP其实不过是Web服务器的一个模块功能，所以首先要保证Web服务器的安全。当然Web服务器要安全又必须是先保证系统安全，这样就扯远了，无穷无尽。常见的web安全漏洞有：注入攻击，跨站攻击，服务器自身漏洞等，对应的详细解释，详见：扩展阅读中的《WEB安全性-2010_OWASP_TOP10》，这里有很详尽的解释。</div>
<div class="para" label-module="para">PHP的优点学习过程和方法</div>
<div class="para" label-module="para">PHP的语法类似于C，Perl，ASP或者JSP。对于那些对上述之一的语言较熟悉的人来说，PHP太简单了。相反的，如果你对PHP了解较多，那么你对于其他几种语言的学习都很简单了。你只需要很短的时间内将PHP的核心语言特点全部掌握，你可能已经非常了解HTML，甚至你已经知道怎样用编辑设计软件或者手工来制作好看的WEB站点。由于PHP代码能够无障碍的添加进你的站点，在你设计和维护站点的同时，你可以很轻松的加入PHP使得你的站点更加具有动态特性。</div>
<div class="para" label-module="para">数据库连接</div>
<div class="para" label-module="para">PHP可以编译成具有与许多数据库相连接的函数。PHP与MySQL是绝佳的组合，如果再加上Apache服务器，就是相当完美的了。你还可以自己编写外围的函数取间接存取数据库。通过这样的途径当你更换使用的数据库时，可以轻松的更改编码以适应这样的变化。PHPLIB就是最常用的可以提供一般事务需要的一系列基库。</div>
<div class="para" label-module="para">可扩展性</div>
<div class="para" label-module="para">就像前面说的那样，PHP<a target=_blank href="/item/%E5%B7%B2%E7%BB%8F">已经</a>进入了一个高速发展的时期。对于一个非程序员来说为PHP扩展附加功能可能会比较难，但是对于一个PHP程序员来说并不困难。</div>

<div class="para" label-module="para">PHP免费安装</div>
<div class="para" label-module="para">PHP源代码包安装版：这个版本适合已经有自己独立的网站域名、网站空间的专业网站建设用户。使用方法依然其为简单，只需三步：</div>

<div class="para" label-module="para">文件格式</div>
<ol class="custom_num para-list list-paddingleft-1"><li class="list-num-1-1 list-num-paddingleft-1"><div class="para" label-module="para">对于只含有 php 代码的文件，我们将在文件结尾处忽略掉 &quot;?&gt;&quot;。这是为了防止多余的空格或者其它字符影响到代码。</div>
</li></ol><div class="para" label-module="para">例如：</div>
<div class="para" label-module="para">$foo = &#39;foo&#39;;</div>
<ol class="custom_num para-list list-paddingleft-1"><li class="list-num-1-1 list-num-paddingleft-1"><div class="para" label-module="para">缩进应该能够反映出代码的逻辑结果，尽量使用四个空格，禁止使用制表符TAB，因为这样能够保证有跨客户端编程器软件的灵活性。</div>
</li></ol><div class="para" label-module="para">例如：</div>

</li></ol><div class="para" label-module="para">例如：</div>
<pre class="brush: php">$variable=&#39;demo&#39;;
$var=&#39;demo2&#39;;

</li><li class="list-num-1-2 list-num-paddingleft-1"><div class="para" label-module="para">每行结尾不允许有多余的空格。</div>
</li></ol><div class="para" label-module="para">Php文件记事本编辑乱码问题</div>
<div class="para" label-module="para">一般情况下，记事本编辑器在对文件进行完编辑并保存之时，其默认编码为ANSI，中文。然则更多的时候，php在语言环境设置时语言多数为utf-8，直接保存并用于apache等http-server解析后就会出现乱码。</div>
<div class="para" label-module="para">为此，应该注意在用记事本编辑完后可将文件用“另存为”的方式对文件进行保存，并将“文件类型”选择“所有文件”，编码与文件指定语言编码一致即可。<sup class="sup--normal" data-sup="4">
